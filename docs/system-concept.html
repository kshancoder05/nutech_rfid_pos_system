Where each function lives

Card (RFID)Identifies a person & pays at the POS; can also tap at the gate.
FingerprintOptional second factor to confirm a payment. Template on ACCOUNT.
FaceHands-free attendance / gate entry; 128-pt descriptor on PEOPLE.
PWAEvery screen. Same app; role-based login changes the experience.
n8nAll logic — validate balance, deduct, generate AccountID, punch states, audit.
AirtableThe one place all data lives; both stations and all viewers share it.

How data flows

  1. A device captures an identity (card UID, face, or fingerprint).
  2. The PWA sends one POST to n8n's webhook.
  3. n8n looks the person up, applies the rule (charge / log a punch / register), writes Airtable.
  4. Admin & Client screens read the same records back — instantly in sync.
  5. Outputs: a printed receipt at the till, the SF2 report at month-end.
